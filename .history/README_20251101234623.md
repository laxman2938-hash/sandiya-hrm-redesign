## Getting Started

### Prerequisites

- Node.js 18+ 
- PostgreSQL database
- Cloudinary account

### Installation

1. Clone the repository:
```bash
git clone <repository-url>
cd sandiya-hrm-redesign
```

2. Install dependencies:
```bash
npm install
```

3. Set up environment variables:
```bash
cp .env.example .env.local
```

Edit `.env.local` with your configuration:
```bash
CLOUDINARY_CLOUD_NAME=your_cloud_name
CLOUDINARY_API_KEY=your_api_key
CLOUDINARY_API_SECRET=your_api_secret
DATABASE_URL="postgresql://username:password@localhost:5432/database_name"
```

4. Set up the database:
```bash
npx prisma generate
npx prisma db push
```

5. Run the development server:

```bash
npm run dev
# or
yarn dev
# or
pnpm dev
# or
bun dev
```

Open [http://localhost:3000](http://localhost:3000) with your browser to see the result.

## Image Upload System

This project uses Cloudinary for image management:

- **Automatic optimization**: Images are automatically compressed and format-optimized
- **CDN delivery**: Global content delivery network for fast loading
- **Folder organization**: Images organized by type (gallery, team, candidates, etc.)
- **Security**: Secure upload and delivery

See [CLOUDINARY_INTEGRATION.md](./CLOUDINARY_INTEGRATION.md) for detailed integration information.

## Development

### Running the Project

Start the development server:
```bash
npm run dev
```

The application will be available at [http://localhost:3000](http://localhost:3000).

### Admin Panel

Access the admin panel at `/admin` to manage:
- Gallery images
- Team members  
- Achievements
- Client information
- Feedback and messages

### API Endpoints

- `POST /api/upload` - Image upload to Cloudinary
- `GET|POST /api/gallery` - Gallery management
- `GET|POST /api/team` - Team member management
- `POST /api/candidates` - Candidate applications

## Deployment

### Vercel Deployment

1. Connect your repository to Vercel
2. Add environment variables in Vercel dashboard:
   - `CLOUDINARY_CLOUD_NAME`
   - `CLOUDINARY_API_KEY`
   - `CLOUDINARY_API_SECRET`
   - `DATABASE_URL`
3. Deploy!

### Environment Variables

Ensure all required environment variables are set:
- Cloudinary credentials for image uploads
- Database connection string
- Any additional API keys
